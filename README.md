Електронний каталог відеоконтенту
(YouTube Clone)
Web-орієнтована інформаційна система для зберігання, каталогізації та перегляду
відеоконтенту. Проєкт реалізує клієнт-серверну архітектуру (Decoupled Architecture) у
вигляді Single Page Application (SPA) та RESTful API.
Технологічний стек
Frontend:
● React 18
● TypeScript
● Vite (збірка)
● Tailwind CSS (стилізація)
● React Router DOM (маршрутизація)
● TanStack Query / React Query (управління серверним станом та кешування)
● Context API (управління глобальним станом авторизації)
Backend:
● PHP 8.x (Native, без фреймворків)
● PDO (безпечна взаємодія з БД через підготовлені запити)
● JWT (JSON Web Tokens для stateless-автентифікації)
База даних та інфраструктура:
● PostgreSQL 15 (реляційна БД)
● Docker & Docker Compose (контейнеризація)
Основний функціонал
● Глобальний каталог: Перегляд списку відео, пошук за назвою та миттєва
фільтрація за категоріями на стороне клієнта.
● Автентифікація та Авторизація: Stateless-система на базі JWT. Паролі хешуються
алгоритмом bcrypt. Захист маршрутів на клієнті (Guard Middleware).
● Історія переглядів: Автоматична фіксація переглянутих відео для авторизованих
користувачів. Використання зв'язків ON DELETE CASCADE для збереження
посилальної цілісності в БД.
● Панель адміністрування: Захищений модуль (RBAC - Role-Based Access Control)

для виконання CRUD-операцій (створення, читання, оновлення, видалення) над
відеоконтентом.
Локальне розгортання (через Docker)
Проєкт повністю контейнеризований. Для запуска необхідний встановлений Docker та
Docker Compose.
1. Клонування репозиторію:
git clone https://github.com/WolandEmort/utube-pet-project.git
cd utube-pet-project

2. Налаштування змінних оточення:
Створіть файл .env у кореневій директорії проєкту:
DB_PASSWORD=your_secure_password
JWT_SECRET_KEY=your_secret_key_for_jwt

Створіть файл .env у директорії frontend/:
VITE_API_URL=localhost:8080

3. Запуск контейнерів:
docker compose up -d --build

4. Доступ до додатку:
● Frontend: localhost:5173
● Backend API: localhost:8080
● База даних (PostgreSQL): порт 5432
Структура бази даних
База даних складається з трьох нормалізованих таблиць:

● users — зберігає дані користувачів, ролі (user, admin) та хеші паролів.
● videos — метадані відеоконтенту (ID, назва, опис, URL прев'ю, категорія).
● view_history — проміжна таблиця (many-to-many), що пов'язує users та videos для
фіксації історії переглядів.
Безпека
● Захист від SQL-ін'єкцій: Використання виключно підготовлених запитів (Prepared
Statements) через абстракцію PDO.
● Захист автентифікації: Перевірка криптографічної сигнатури токенів
(HMAC-SHA256) на кожному приватному маршруті.
● CORS: Налаштована політика крос-доменних запитів на стороні бекенду.
Розроблено як Pet-проєкт для демонстрації навичок Fullstack-розробки.
